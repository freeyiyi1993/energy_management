# 精力管理可视化 - 产品需求文档 (PRD)

## 1. 项目背景

现代知识工作者面临的核心挑战不是时间管理，而是**精力管理**。精力是有限的、可恢复的、可量化的资源。本项目通过游戏化机制，帮助用户直观感知自己的身体精力和思维精力状态，养成健康习惯，提升专注效率。

## 2. 项目目标

- 让用户像看「血条」一样实时感知精力状态
- 通过打卡机制引导健康习惯（睡眠、运动、饮食、饮水等）
- 用番茄钟管理专注时间，追踪专注质量
- 通过「完美一天」奖惩机制激励长期坚持
- 支持 Chrome 扩展 + Web 双端使用，数据跨设备同步

## 3. 目标用户

知识工作者、程序员、远程办公者 —— 长时间在电脑前工作，需要主动管理精力和习惯。

## 4. 核心功能

### 4.1 精力条系统

| 功能 | 说明 |
|------|------|
| 精力值显示 | 实时精力百分比 + 颜色渐变（绿→橙→红） |
| 自然衰减 | 每分钟按 `decayRate/60` 衰减，模拟精力消耗 |
| 餐食惩罚 | 10:00 后未吃 1 餐, 14:00 后未吃 2 餐, 19:00 后未吃 3 餐，衰减乘以 `penaltyMultiplier` |
| 低精力提醒 | 精力 <20% 全屏警告，每天仅提醒一次 |
| 睡眠恢复 | 睡眠设定当天精力起点，扣除已消耗部分：`max(当前精力, 睡眠起点 - 今日已消耗)`。打卡越早恢复越多 |
| 每日重置 | 每天 8:00 AM 重置，精力值由睡眠时长决定 |

### 4.2 打卡任务

| 功能 | 说明 |
|------|------|
| 内置任务 (8个) | 睡眠、运动、三餐、饮水、拉伸、午睡、冥想、肠道 |
| 自定义任务 | 用户可新增/编辑/删除，设置恢复等级和类型 |
| 任务类型 | counter（计数器）、boolean（开关）、number（数值输入） |
| 恢复等级 | none / small / mid / big，对应不同精力恢复量 |
| 完美一天标记 | 每个任务可设置是否计入「完美一天」判定 |

### 4.3 番茄钟

| 功能 | 说明 |
|------|------|
| 25 分钟计时 | 圆形进度条倒计时 |
| 专注评分 | 完成后滑动评分 0-100%，100% 为完美番茄 |
| 连续限制 | 连续 3 个番茄后强制休息提醒 |
| 统计追踪 | 记录总数、完美数、连续数 |

### 4.4 长期成长系统

| 功能 | 说明 |
|------|------|
| 完美一天 | 所有必要任务(计入完美一天的任务)完成 + 4 个完美番茄 → 次日 maxEnergy +1 |
| 糟糕一天 | 0 完美番茄 + 运动 <30min + 睡眠 <6h → 次日 maxEnergy -1 |
| 节假日豁免 | 若当天没有任何日志, 默认节假日模式, 次日不扣精力上限 |
| 精力上限 | maxEnergy 随长期表现浮动，可视化成长曲线 |

### 4.5 统计与日志

| 功能 | 说明 |
|------|------|
| 折线图 | maxEnergy / energyConsumed / 番茄数 / 完美番茄数 趋势 |
| 操作日志 | 按日期分组，可展开查看每条记录 |
| 紧凑存储 | 日志用 `[timestamp, actionType, value, energyDiff]` 压缩 |

### 4.6 跨设备同步

| 功能 | 说明 |
|------|------|
| Google 登录 | 扩展用 chrome.identity，Web 用 signInWithPopup |
| 自动同步 | 登录后自动拉取，每 60 秒自动推送 |
| 手动操作 | 拉取（智能合并）、强制拉取（云端覆盖）、推送 |
| 离线可用 | 未登录时完全本地运行，登录后无缝同步 |

### 4.7 设置页

| 功能 | 说明 |
|------|------|
| 参数配置 | maxEnergy / decayRate / penaltyMultiplier 等 |
| 任务管理 | 启用/禁用/编辑/删除任务，内置任务标记了完美一天的不可删, 其余可删 |
| 任务编辑器 | 弹窗表单：图标、名称、类型、恢复等级等 |

### 4.8 规则说明页

| 功能 | 说明 |
|------|------|
| 精力规则 | 每日恢复、衰减、餐食惩罚、完美/糟糕一天的判定逻辑 |
| 同步规则 | 登录后自动拉取、每 60 秒自动推送、拉取/强制拉取/推送的区别 |
| 动态内容 | 根据当前启用的任务动态显示判定条件 |

## 5. 双端差异

| 维度 | Chrome 扩展 | Web 版 |
|------|------------|--------|
| 入口 | 浏览器工具栏弹窗 (popup) | 独立网页 |
| 后台 | Service Worker + chrome.alarms | setInterval (web-ticker) |
| 存储 | chrome.storage.local + firebase | localStorage + firebase |
| 登录 | chrome.identity OAuth | Firebase signInWithPopup |
| 提醒 | 新开标签页（全屏提醒） | 当前页全屏提醒 |
| 样式 | 卡片式（固定宽度弹窗） | 平铺式（响应式容器） |

## 6. 用户故事

### 日常使用
- 作为用户，我打开扩展弹窗看到当前精力值，知道自己还剩多少"电量"
- 作为用户，我完成一项任务后点击打卡，精力恢复，获得正反馈
- 作为用户，我启动番茄钟专注工作，结束后给自己打分

### 习惯养成
- 作为用户，我通过完美一天机制被激励完成每日任务
- 作为用户，我查看统计页的趋势图，了解长期精力变化
- 作为用户，我自定义打卡项目（如臀桥），适配个人习惯

### 跨设备
- 作为用户，我在公司用扩展打卡，回家用 Web 版继续，数据自动同步
- 作为用户，我换了电脑后登录就能恢复所有数据

### 设置
- 作为用户，我调整衰减速率让精力下降更符合我的体感
- 作为用户，我禁用不需要的内置任务（如冥想）

## 7. 不做什么（边界）

- **不做社交功能** —— 这是个人工具，不做排行榜、好友互动
- **不做多用户** —— 一个 Google 账号对应一份数据
- **不做原生 App** —— 只做 Chrome 扩展 + Web PWA
- **不做 AI 建议** —— 不根据数据自动推荐行为
- **不做通知推送** —— 扩展用标签页提醒，不用系统通知
- **不做历史数据编辑** —— 日志只追加，不可修改
- **不做数据导出** —— 暂不提供 CSV/JSON 导出功能
- **不做国际化** —— 界面仅中文
